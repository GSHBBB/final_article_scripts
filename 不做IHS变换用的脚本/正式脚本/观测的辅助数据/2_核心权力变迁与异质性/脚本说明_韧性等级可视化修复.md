# 韧性等级可视化修复脚本说明

## 脚本名称
`修复_韧性等级可视化_v2.py`

## 修复内容

### 问题描述
原始的韧性等级可视化存在以下问题：
1. **国家聚合体未过滤** - 在可视化中包含了世界银行的区域聚合体代码
2. **香港标签不规范** - 香港仍使用代码"HKG"或"HK"，未改为中英双语标签"Hong Kong (China)"

### 解决方案
脚本在**仅在画图部分进行过滤**，不修改原始输入数据文件：

#### 1. 国家聚合体过滤
- **过滤范围**：World Bank宏观区域代码（ARB, WLD等）和其他聚合体代码
- **过滤方式**：
  ```
  - 原始韧性数据：3946行，198个国家
  - 过滤后数据：3946行，198个主权国家
  ```
- **注意**：当前输入数据已经相对清洁，聚合体已在上游处理

#### 2. 香港标签修正
- **原始代码**：HKG (香港的ISO 3-letter代码)
- **修正标签**：Hong Kong (China)
- **验证表结果**：
  ```
  Row 5: Hong Kong (China), 超级枢纽, 中心度1.0, 韧性11.30
  ```

### 技术细节

#### 过滤机制
```python
# 世界银行宏观区域代码（示例）
WB_MACRO_CODES = {
    'ARB', 'CSS', 'CEB', 'EAR', 'EAS', 'EAP', 'EMU', 'ECA', 'ECS', 'EUU',
    ...
    'UMC', 'WLD', 'WXD'
}

# 其他聚合体
OTHER_AGGREGATES = {'EU27_2020', 'OECD', 'G20', 'BRICS'}

# 过滤条件
resi_filtered = resilience_df[
    ~resilience_df['REF_AREA'].isin(WB_MACRO_CODES) &
    ~resilience_df['REF_AREA'].isin(OTHER_AGGREGATES)
].copy()
```

#### 韧性分层
- **方法**：按年份单独分层，每年一个qcut分组
- **分层标准**：出度中心度（Out_Degree_Centrality）
- **分层结果**：4个等级
  - Tier_1_Edge（边缘节点）：中心度最低25%
  - Tier_2_Main（主要节点）：中心度25%-50%
  - Tier_3_Core（核心枢纽）：中心度50%-75%
  - Tier_4_Super（超级枢纽）：中心度最高25%

#### 香港标签修正
